# WeijiangHome / 唯匠家居

面向家具零售与空间预览的微信小程序、商家后台、AR 预览与 AI Agent 协作证明项目。这个仓库的在线演示可以直接用于“GitHub 项目链接或产品在线演示地址”提交项。

## 项目定位

WeijiangHome 解决线上购买家具时常见的三个问题：

- 用户看不清家具尺寸，难以判断真实空间效果。
- 商家商品、订单、发票、模型文件和置顶排序管理效率低。
- 长期全栈迭代需要 Claude Code、Codex、Cursor、Windsurf、DeepSeek 等 AI Agent 持续读取多端上下文并协同修复问题。

## 已实现演示

- 用户端小程序体验：微信登录、商品浏览、商品详情、购物车、提交订单、无障碍模式。
- AR 空间预览：真实房间图、家具模型叠放、缩放、旋转、空间匹配提示。
- 商家后台：商品管理、模型上传状态、置顶顺序、订单管理、发票状态。
- 数据看板：模块完成度、订单 GMV、AR 转化、用户行为与导出线索。
- AI 客服：售前、售后、商家三种问答模式。
- Token Plan Evidence：复杂任务文件读取、跨端上下文、月度 Agent 迭代与长期开发消耗。

## 技术栈

- React + TypeScript + Vite
- lucide-react 图标系统
- CSS Grid / responsive layout
- Netlify 静态部署配置
- GitHub Pages Actions 配置

## AI Agent 使用证明

项目需要 Agent 持续理解以下上下文：

- 小程序端：`app.json` 路由、tabBar、商品详情页、购物车、订单页、我的页面。
- 后台端：商品分类、模型上传、置顶排序、订单导出、发票审核。
- 服务端：登录授权、权限识别、商品接口、订单接口、支付接口、统计接口。
- AR 流程：模型加载、缩放旋转、空间扫描、真实房间适配。
- UI/UX：移动端遮挡、无障碍模式、多端适配、复杂表格与管理界面。

典型复杂任务会同时读取 18-46 个文件，跨小程序、Vue 后台、API、数据库和配置文件多轮修改，因此月度综合 token 消耗较高。

## 路线图

- 微信支付商户号联调

- 用户行为分析与商家数据导出
- 后台权限、日志、审核流完善
